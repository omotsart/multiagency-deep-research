# Deep Research

Автономный агент-исследователь на **OpenAI Agents SDK** с интерфейсом на **Gradio**.
Пользователь задаёт тему → агент уточняет детали → автономно исследует веб →
пишет отчёт с самопроверкой качества → показывает результат.

Проект развивает учебный базовый пример (последовательный вызов агентов) в
сторону настоящей агентной системы с автономией, контролем качества и
человеком в цикле.

## Возможности

- Уточняющие вопросы перед исследованием (человек в цикле).
- Автономный менеджер, сам решающий число и направление поисков.
- Параллельный веб-поиск.
- Контроль качества отчёта по паттерну Evaluator–Optimizer.
- Пошаговый интерфейс с живым прогрессом.

## Архитектура (три зоны)

**Прелюдия (человек в цикле).** `clarifier_agent` задаёт 3 вопроса; ответы
склеиваются с темой в единый бриф — источник намерения для всего пайплайна.

**Автономное ядро.** `manager_agent` вызывает специалистов как инструменты
(agents-as-tools): планировщик, параллельный поиск, писатель. Сам решает, сколько
и каких поисков провести, включая динамическую вторую волну по пробелам в данных.

**Контроль качества.** `evaluator_agent` оценивает отчёт; детерминированная петля
в коде дорабатывает его до порога качества либо до лимита итераций.

### О паттернах: agents-as-tools и handoffs

Основной паттерн оркестрации — **agents-as-tools**: менеджер остаётся владельцем
процесса и финального результата, вызывая агентов как инструменты. Это осознанный
выбор — согласно документации OpenAI, конвейеры обработки (а это именно наш случай:
тема → план → поиск → отчёт → оценка) ложатся на agents-as-tools лучше, чем на
handoffs, и проще отлаживаются.

**Handoff** используется точечно в одном месте — на финальном оформлении отчёта,
после утверждения его оценщиком. Это демонстрирует владение обоими паттернами,
не нарушая контроль менеджера над основной работой.

## Стек

Python 3.10+, OpenAI Agents SDK (`openai-agents`), Pydantic, Gradio, `asyncio`.

## Запуск локально

```bash
python -m venv .venv
# Windows: .venv\Scripts\activate | macOS/Linux: source .venv/bin/activate
pip install -r requirements.txt

cp .env.example .env   # затем впиши OPENAI_API_KEY
python app.py
```

## Тесты

```bash
python -m pytest
```

Тесты используют моки LLM — без реальных вызовов OpenAI и без ключа.

## Структура

```
app.py            точка входа (Gradio)
src/              агенты и оркестрация
tests/            тесты на моках
docs/             документы процесса разработки (WORKFLOW, DECISIONS, RULES, …)
handover/         handover-документы по gates
```

## Документация процесса

Разработка ведётся по gates с контролем качества. Ключевые документы:
`docs/WORKFLOW.md` (карта gates), `docs/DECISIONS.md` (архитектурные решения),
`docs/RULES.md` (инварианты проекта), `CLAUDE.md` (контекст для Claude).

## Цель gate
Менеджер-агент оркестрирует инструменты сам (agents-as-tools, D-01): принимает
решения о числе и направлении поисков, включая динамическую вторую волну по
пробелам в данных. Ядро зоны 2 из CLAUDE.md.

## Критерии закрытия (чек-лист дословно из WORKFLOW.md)
- [ ] `manager_agent` с инструментами: план, поиск, писатель.
- [ ] Промпт менеджера: бюджет поисков, инструкция про вторую волну по пробелам.
- [ ] E2E happy-path ЗЕЛЁНЫЙ: `бриф → план → поиски → отчёт` на моках.
- [ ] Юнит-тест: менеджер уважает бюджет поисков (не превышает лимит).

**Тесты:** unit (соблюдение бюджета) + E2E (контракт трубы).
**Не тестируем:** конкретные решения LLM о запросах — недетерминированны.

## НЕ сделано / точка остановки
Пять пунктов входа из SESSION_HANDOVER §8, ещё НЕ выполнены:
1. **Подключение предохранителей к рантайму** (3b). Без этого 2d — мёртвый код:
   бюджет не читается, `max_turns` не действует. E2E-скелет это проверяет.
2. **Оживление E2E** (3c). Создать менеджера → снять `xfail`. Скелет НЕ
   переписывать, только оживить.
3. **DEFERRED #4** — двойник RunResult в мок-слое при извлечении вывода
   `.as_tool()`-обёрток самим SDK, когда их дёргает менеджер. Решается при
   оживлении трубы (3c).
4. **Граница D-09** — где ложится handoff на `finalizer_agent`. Критическое
   решение владельца (RULES §1), НЕ решаю сам. Приносится на шаге 3d. Надзор
   (владелец) заранее отметил: D-09 логически завязан на evaluator-петлю (Gate 4),
   поэтому полноценный финализатор физически может лечь на стык Gate 3 / Gate 4.
5. **Тест соблюдения бюджета менеджером** (3b) — юнит-тест, что менеджер не
   превышает лимит поисков.

**Пункт внимания к 3b (f-string бюджета).** Промпт менеджера берёт бюджет жёстко
из `DEFAULT_SEARCH_BUDGET` через f-string. Если на 3b `ResearchContext`
создаётся с нестандартным лимитом, текст промпта разъедется с реальным бюджетом
(предохранитель в коде сработает верно, но менеджеру будет обещано другое число).
Решить на 3b: либо оба значения от одной константы, либо прокидывать реальный
лимит в промпт динамически.

## Цель gate
Первый агент (`clarifier_agent`) задаёт ровно 3 уточняющих вопроса по теме;
ответы детерминированно склеиваются с темой в единый бриф. Живёт ВНЕ автономного
ядра (Зона 1, человек в цикле).

## Критерии закрытия (чек-лист из WORKFLOW.md)
- [x] `clarifier_agent` с `output_type` на ровно 3 вопроса (Pydantic).
- [x] Функция склейки `тема + Q&A → brief` (детерминированная, чистая).
- [x] Юнит-тест склейки брифа (контракт передачи данных).
- [x] Юнит-тест валидности Pydantic-схемы вопросов.

## Сделано в этой сессии
- `src/clarifier_agent.py`: агент `ClarifierAgent`, модель `gpt-4o-mini`,
  `output_type=ClarifyingQuestions`. Схема требует ровно `NUM_QUESTIONS = 3`
  вопроса через `Field(min_length=3, max_length=3)`. Стиль согласован с
  `planner_agent.py`/`writer_agent.py`.
- `src/brief.py`: чистая функция `build_brief(topic, questions, answers) -> str`.
  Без LLM/окружения/побочных эффектов. Нумерованные пары Q&A, обрезка пробелов,
  плейсхолдер для пустого ответа, валидация (пустая тема / рассинхрон длин → `ValueError`).
- `tests/test_brief.py`: 8 юнитов (тема в брифе, порядок пар, обрезка, плейсхолдер,
  только тема без вопросов, детерминизм, пустая тема, рассинхрон длин).
- `tests/test_clarifier_schema.py`: 4 юнита (NUM_QUESTIONS == 3, ровно 3 — валидно,
  2 и 4 — отклоняются `ValidationError`).
- `pytest.ini`: `pythonpath = src`, `testpaths = tests` (импорт тестов без sys.path-хаков).
- Прогон: **12 passed**. Импорт `clarifier_agent` не требует ключа OpenAI (реальных вызовов нет).
- Кэши (`__pycache__`, `.pytest_cache`) вычищены после прогона.

## Решения, требующие владельца
- Формат брифа выбран как **плоская строка** (`str`) с нумерованными парами Q&A —
  реализация в рамках scope Gate 1 («строку/структуру» по WORKFLOW). Это фактический
  контракт данных, который будет потреблять менеджер на Gate 3. Если потребуется
  структурированный бриф (Pydantic) — это правка контракта уровня владельца
  (RULES §1), заносим отдельным решением в `DECISIONS.md`. Пока помечаю как
  **ПРЕДЛОЖЕНО (по умолчанию: строка)**, менять на Gate 1 не требуется.
- Открытых блокирующих вопросов нет. Q-01 закрыт ранее (D-09).
